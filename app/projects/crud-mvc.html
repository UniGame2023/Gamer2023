CRUD MVC

Nome do Projeto: Sistema de Gerenciamento de Usuários (CRUD)

Tecnologias Utilizadas:

  • Linguagem de Programação: PHP
  • Linguagens de Marcação: HTML
  • Folhas de Estilo: CSS
  • Padrão de Projeto: Modelo-Visão-Controlador (MVC)
  • Funcionalidades Específicas: Rotas e URL Amigável

Descrição: O projeto é um Sistema de Gerenciamento de Usuários desenvolvido para facilitar o cadastro, visualização, edição e exclusão de registros de usuários. Utilizando uma combinação de PHP, HTML e CSS, o sistema proporciona uma experiência amigável e funcional para o usuário final.

Funcionalidades Principais:

  • Cadastro de Usuários:
    Os usuários podem ser cadastrados no sistema, inserindo as seguintes informações:
    nome completo do usuário, endereço de e-mail do usuário, data de nascimento do usuário.
    Os dados fornecidos pelos usuários são armazenados em um banco de dados para futura referência.
  • Listagem de Registros:
    O sistema exibe uma tabela que lista todos os registros de usuários cadastrados.
    Cada registro exibe as informações do usuário, incluindo nome, e-mail e data de nascimento.
  • Edição de Registros:
    Os usuários têm a opção de editar os registros existentes.
    Ao selecionar a opção de edição, eles podem atualizar o nome, o e-mail ou a data de nascimento do usuário.
  • Exclusão de Registros:
    Os usuários têm a capacidade de excluir registros de usuários existentes.
    Ao selecionar a opção de exclusão, o registro correspondente é removido permanentemente do banco de dados.
  • Arquitetura MVC: O projeto segue o padrão de projeto Modelo-Visão-Controlador (MVC), que ajuda a manter
    uma estrutura organizada e facilita a manutenção e expansão do sistema. Nesse padrão:
    • Modelo (Model): Lida com a lógica de negócios e a interação com o banco de dados. É responsável por armazenar e recuperar os dados dos usuários.
    • Visão (View): É responsável por exibir os dados aos usuários de forma amigável e interativa. No contexto do projeto, a "visão" pode ser a página web que exibe a tabela de registros e formulários para edição e exclusão.
    • Controlador (Controller): Controla o fluxo de interação entre a visão e o modelo. Lida com as requisições dos usuários, processa os dados e coordena as operações de criação, leitura, atualização e exclusão de registros.
  • URL Amigável e Rotas:
    O projeto utiliza URLs amigáveis e rotas para criar uma experiência de usuário mais intuitiva. Isso significa que as URLs são projetadas de forma legível e significativa, como "/editar?id=1" para editar o usuário com o ID 1.

Conclusão:
O Sistema de Gerenciamento de Usuários é uma aplicação que utiliza tecnologias web padrão e um padrão de projeto MVC para fornecer funcionalidades completas de CRUD. Ele permite aos usuários cadastrar, visualizar, editar e excluir registros de usuários de maneira eficiente e amigável. Essa aplicação é útil para gerenciar informações de usuários em diversos contextos, como registros de clientes, funcionários ou membros de um site.
